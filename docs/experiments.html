RAG 工作坊作品集 比較與評估：每個功能至少一個 A/B
總覽 HR 版 教學設計 比較與評估 技術細節 週誌
原則：不要只看答案好不好看，要看「指標」

這些比較，學生做完就能知道差在哪

平台會輸出評估結果（CSV/JSON）。你也可以把 JSON 轉成 HTML 報告，直接貼到作品集或交作業。

必做 A/B（每個功能至少一個）

  • Top‑K：3 vs 10（看 HitRate、Recall、chunk 數量、延遲）
  • 重寫：off vs on（看檢索命中與答案貼題程度）
  • 圖譜：off vs on（看 evidence、關係路徑、來源引用）
  • Agentic：L1 vs L2（看補問/再檢索、來源強制）
  • Agentic：L2 vs L3（看工具行動記錄、多跳推理）

指標怎麼解讀？（台灣用語版）

指標 意思（白話） 你要判斷什麼
Hit / HitRate 有沒有撈到「該撈的」文件（至少有命中） 命中率變高通常是好事，但要注意是不是撈太多
Term / Source Recall 期望關鍵字/期望來源有沒有被撈進來 用來看重寫、Top‑K、圖譜有沒有真的幫到檢索
Score（avg/top） 檢索分數（相似度/打分） Top‑K 大時 avg 可能下降，但 top 不一定降
Elapsed ms 每題花多久 成效提升要值得時間成本

把 JSON 變成一份「可以交」的報告

教學坊下載的 rag-eval-*.json 可以用工具轉成 HTML，放到 docs/reports/。

# 1) 在「RAG 教學坊」下載 eval JSON
# 2) 產生報告（輸出到 docs/reports/）
node ./docs/tools/export-eval-report.mjs --in ./rag-eval-demo.json --out ./docs/reports/demo.html
提醒：這份報告給 HR/主管最有用的是「你做了哪些比較」跟「每個比較的指標差異」。

想更像研究/工程？

我建議你把比較寫成這樣

「設定 A：Top‑K=3、重寫=off；設定 B：Top‑K=10、重寫=on。B 的 hitRate +12%，但平均延遲 +180ms。結論：在這組資料，B 比較適合長問句。」
