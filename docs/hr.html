我做了一個「能教、能比、能驗證」的 RAG 平台

我把 RAG 變成一個教學型產品：學生不用只聽概念，而是直接在頁面上做 A/B 比較，最後能把結果匯出成報告。

看 GitHub

這個專案在解什麼問題？

RAG 教學常遇到兩件事：學生聽得懂名詞，但不知道「調參」到底會怎麼影響答案；以及做完也很難留下可驗證的成果。

我把它變成：每個功能都可比較（A/B），每次比較都能看到差異指標，最後能輸出報告。

你會在平台看到什麼？

  • 聊天：可用自己的知識庫回答，並（可選）強制附來源。
  • 知識庫：上傳文件、看 chunks、管理檔案、看知識圖譜。
  • RAG 教學坊：同一題做 Top‑K / 重寫 / 圖譜 / Agentic 的比較。
  • 學生儀表板：把練習建議、題目、觀察點整理在同頁。

我如何證明我不是「只有做 UI」？

項目 有什麼可驗證的輸出 你看得到什麼
RAG A/B 比較 評估 CSV/JSON HitRate、Recall、Score、時間等指標
知識圖譜 節點/關係/證據 API 圖上的每條邊都有證據來源
教學流程 可重做的練習清單 學生自己調參、觀察、寫結論

如果你要我現場 demo

我會怎麼 demo（3 分鐘版）

1) 管理員上傳文件 → 2) 去 RAG 教學坊拿同一題做 A/B → 3) 匯出 eval JSON → 4) 用工具轉成 HTML 報告。

我希望你注意的地方
  • 不是只看「答案好不好看」，而是看「有沒有來源」、「指標有沒有變好」。
  • 同一題用不同設定，差異是可複製、可寫進報告的。
